What is a "gazetteer"?

Most people may think of "gazette," as in a newspaper, but a gazetteer is actually completely different. As early as Ancient Greece, gazetteers were publications that contained geographic and demographic information about a location, essentially a geographic dictionary or almanac.

Gazetteer HK is a modern day version of this concept. We collect a variety of data sources into a single unified and standardized database, available for anyone to use for whatever the use case.

See the Wikipedia entry in English and Chinese for more history about gazetteers.

How do I use this?

If you're interested in seeing visualizations of the data, check out some of our pre-made apps, such as the Neighborhood Profiles, or the Data Mapper.

If you're interested in using the API, check out the Data Browser for documentation, and visit our GitHub page for the source code.

What kind of use cases are there for Gazetter HK?

  • If you're a journalist, Gazetteer's Census data makes it easy for you to look up background information about a district or neighborhood that you're writing a story on.
  • If you're a researcher, you can download our full dataset and join it to your own data for your analysis
  • If you're in marketing, you can use our our neighborhood profiles to find out which areas to target
  • If you're building an app that requires showing Hong Kong's districts or areas, you can use our GeoJSON files to draw the map
  • Or maybe you just like open data and cool visualizations (who doesn't?)

Where did you get your data?

The Hong Kong Government has made publicly available a number of official data sets through the Data.One project. The 2011 Census dataset makes up the bulk of our data, but we also use a few others, such as the public facilities dataset. We're also continuing to integrate more datasets into the API.

How much does it cost to use the API?

Nothing! At the moment, we don't have plans to charge for API usage. Our project is also fully open source, and contains the scripts necessary to generate our full dataset from scratch.

Can I help contribute?

Absolutely. Check out some of the issues on our issue tracker, fork the code on Github and send a pull request. If you have any questions, get in touch with us at contact@gazetteer.hk.

What's your stack?

Our backend API was built with Python using Flask. Flask interacts directly with a pandas DataFrame to query the data, so there is no database involved.

Our frontend is built primarily using AngularJS, Leaflet, and dimple, which builds on top of the D3js framework. For translations, we use i18next. There are also a handful of other plugins and utilities that we use. Check out our bower file for full details.
